Book Editor Studio — Estabilización & Motor de Traducción Inteligente

Cierre de hito: recuperación de datos perdidos, corrección de arquitectura y lanzamiento del sistema de auto-traducción con Dirty-Checking.

1 · Resumen Ejecutivo

El módulo Book Editor Studio (book_editor.php) presentaba dos secciones renderizando vacías: las Seven Curiosity Cards y el Duane Hallock Master Testimonial. La causa raíz no era una base de datos vacía, sino un desajuste de nomenclatura entre el editor y los datos reales ya almacenados en producción. Se corrigió el mapeo, se recuperó el contenido con datos de respaldo oficiales, y se construyó un botón de auto-traducción EN→ES que respeta las ediciones manuales existentes.

2 · Causa Raíz y Recuperación de Datos

  • book_editor.php y api/book_editor.php leían/escribían el meta_key testimonial, pero la tabla real lly_book_content ya contenía datos vivos bajo testimonial_quote y testimonial_author — este último sin ningún input en el formulario para editarlo.
  • Las 7 Curiosity Cards solo tenían fila real para card_6 (placeholder de prueba: "Test" / "Prueba"); las cards 1–5 y 7 nunca se guardaron.
  • Se agregó un sistema de fallback bilingüe oficial en el editor: si el DB no tiene fila o viene vacía para un idioma, el input se pre-llena con el texto oficial (nunca queda un layout vacío), respetando siempre los valores reales de BD cuando existen.
  • Se conectó por primera vez el campo testimonial_author al formulario del editor y a la página pública book.php (antes hardcodeado en el HTML, ahora editable en vivo).

3 · Motor de Traducción Inteligente Multilingüe (Dirty-Checking)

Nuevo botón "🌐 Translate Missing Fields" en book_editor.php, respaldado por el endpoint api/translate.php. Sigue tres reglas de control de cambios para nunca corromper una traducción manual:

  1. Español vacío → se traduce el inglés correspondiente.
  2. Inglés cambió respecto al valor cargado originalmente de la BD → se re-traduce y sobreescribe el español.
  3. Español ya tiene texto y el inglés no cambió → se deja intacto (protege traducciones manuales).

Proveedor: Google Translate (endpoint público client=gtx), migrado desde DeepL el mismo día por bloqueo regional de registro de cuenta — sin necesidad de API key ni configuración en .env. Toda la llamada ocurre en el backend; el navegador nunca contacta al proveedor directamente.
